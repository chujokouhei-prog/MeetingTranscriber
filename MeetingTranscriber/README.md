# MeetingTranscriber

MeetingTranscriber は、打ち合わせ音声を録音し、録音ごとに文字起こし結果を管理する SwiftUI 製 iOS アプリです。

個人検証用のアプリとして作っています。録音ファイルと文字起こし結果はアプリ内に保存し、OpenAI API などの外部 API は使用していません。文字起こしには Apple 標準の Speech フレームワークを使用します。

## できること

- 打ち合わせ音声を録音する
- 録音前に、お客様本人が確認して同意できる画面を表示する
- 録音ファイルをアプリ内の Documents フォルダに `.m4a` 形式で保存する
- 保存済みの録音ファイル一覧を表示する
- 録音一覧で、録音ごとの自動削除までの残り時間を表示する
- 録音ファイルを再生、一時停止、シークする
- Apple 標準の Speech フレームワークで日本語の文字起こしをする
- 文字起こし結果を録音ファイルごとに保存する
- 文字起こし結果をコピーする
- 録音名を変更する
- 録音一覧の横スライド操作から、名称変更と削除を行う
- 削除前に確認ダイアログを表示する
- 録音ファイルを手動削除し、関連する文字起こし結果も削除する
- アプリを閉じて開き直しても、保存済みの録音と文字起こし結果を表示する
- 作成から 24 時間以上経過した録音ファイルを自動削除する
- 作成から 24 時間以上経過した文字起こし結果を自動削除する

## まだできないこと

- 文字起こし結果の要約
- 文字起こし結果の編集
- 録音ごとのメモ管理
- 録音ファイルや文字起こし結果のエクスポート
- クラウド同期
- 複数端末でのデータ共有
- 外部 API を使った高精度な文字起こし

## 使用している主な技術

- Swift
- SwiftUI
  - 画面表示、録音一覧、詳細画面、同意画面に使用
- AVFoundation
  - 録音、再生、一時停止、シークに使用
- Speech
  - Apple 標準の音声認識に使用
- UserDefaults
  - 文字起こし結果の保存に使用
- FileManager
  - 録音ファイルの保存、名称変更、削除、自動削除に使用

## ローカルでの起動方法

1. Xcode で `MeetingTranscriber.xcodeproj` を開きます。
2. 実行先に iPhone シミュレータ、または実機を選びます。
3. Xcode の Run ボタンを押して起動します。

マイク録音や音声認識を使うため、Info 設定に以下の権限説明が必要です。

- `Privacy - Microphone Usage Description`
  - 例: `音声を録音するためにマイクを使用します`
- `Privacy - Speech Recognition Usage Description`
  - 例: `録音した音声を文字起こしするために音声認識を使用します`

バックグラウンド録音のため、`Info.plist` に `UIBackgroundModes` の `audio` を設定しています。

## データ保存方針

録音ファイルは、アプリ内の Documents フォルダに `.m4a` 形式で保存します。

文字起こし結果は、録音ファイルに紐づけて UserDefaults に保存します。

このアプリでは、録音データや文字起こし結果を独自のサーバーやクラウドへ保存しません。また、OpenAI API などの外部 API も使っていません。

ただし、文字起こしには Apple 標準の Speech フレームワークを使用します。端末や設定によっては、Apple の音声認識機能の動作条件に依存します。

## 削除方針

録音ファイルは、作成日時から 24 時間以上経過すると自動削除されます。

文字起こし結果も、作成日時から 24 時間以上経過すると自動削除されます。

また、対応する録音ファイルが削除された場合、その録音ファイルに紐づく文字起こし結果も削除されます。

録音一覧から手動で削除した場合も、録音ファイル本体と関連する文字起こし結果を削除します。

## 画面構成

- 録音一覧画面
  - 録音開始、録音停止
  - 録音状態の表示
  - 保存済み録音の一覧
  - 横スライドによる名称変更、削除
- 録音詳細画面
  - 録音情報
  - 再生、一時停止、シーク
  - 文字起こし
  - 文字起こし結果の表示とコピー
- 同意画面
  - お客様本人が録音内容を確認し、同意して録音を開始する画面
