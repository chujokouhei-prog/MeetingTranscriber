README.md
# MeetingTranscriber

MeetingTranscriber は、打ち合わせ音声を録音し、端末内で文字起こしするための SwiftUI 製 iOS アプリです。

まずは個人検証用のアプリとして作っています。録音データや文字起こし結果を外部 API に送信しない方針です。

## できること

- 打ち合わせ音声を録音する
- 録音ファイルを端末内の Documents フォルダに保存する
- 保存済みの録音ファイル一覧を表示する
- 録音ファイルを再生する
- Apple 標準の Speech フレームワークで日本語の文字起こしをする
- 文字起こし結果を録音ファイルごとに保存する
- アプリを閉じて開き直しても、保存済みの文字起こし結果を表示する
- 作成から 24 時間以上経過した録音ファイルを自動削除する
- 作成から 24 時間以上経過した文字起こし結果を自動削除する

## まだできないこと

- 文字起こし結果の要約
- 文字起こし結果の編集
- 録音ファイルや文字起こし結果の手動削除
- 録音ファイルの名前変更
- クラウド同期
- 外部 API を使った高精度な文字起こし
- 複数端末でのデータ共有

## 使用している主な技術

- Swift
- SwiftUI
- AVFoundation
  - 録音と再生に使用
- Speech
  - Apple 標準の音声認識に使用
- UserDefaults
  - 文字起こし結果の保存に使用

## ローカルでの起動方法

1. Xcode で `MeetingTranscriber.xcodeproj` を開きます。
2. 実行先に iPhone シミュレータ、または実機を選びます。
3. Xcode の Run ボタンを押して起動します。

マイク録音や音声認識を使うため、Info 設定に以下の権限説明が必要です。

- `Privacy - Microphone Usage Description`
  - 例: `打ち合わせの音声を録音するためにマイクを使用します`
- `Privacy - Speech Recognition Usage Description`
  - 例: `録音した打ち合わせ音声を文字起こしするために音声認識を使用します`

## データ保存方針

録音ファイルは、アプリ内の Documents フォルダに `.m4a` 形式で保存します。

文字起こし結果は、録音ファイル名と紐づけて UserDefaults に保存します。

このアプリでは、録音データや文字起こし結果をクラウドへ保存しません。また、OpenAI API などの外部 API も使っていません。

## 24時間で削除する方針

録音ファイルは、作成日時から 24 時間以上経過すると自動削除されます。

文字起こし結果も、作成日時から 24 時間以上経過すると自動削除されます。

また、対応する録音ファイルが削除された場合、その録音ファイルに紐づく文字起こし結果も削除されます。

## 今後追加したい機能

- 文字起こし結果の要約
- 文字起こし結果の編集
- 録音ファイルの手動削除
- 録音ファイル名の変更
- 文字起こし結果のコピー
- 録音中の経過時間表示
- より見やすい録音詳細画面
